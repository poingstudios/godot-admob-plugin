# テスト広告を有効にする

!!! note "Godot 3 (v1) ドキュメント"
    このページは **Godot 3.x** 用です。**Godot 4.2+** については、[安定版ドキュメント](https://poingstudios.github.io/godot-admob-plugin/stable/) をご覧ください。

開発中は、無効なトラフィックを生成し、AdMobアカウントの停止リスクを避けるために、テスト広告を使用することが重要です。

---

## 1. エディタでのモック広告のプレビュー

プラグインには内蔵のモック広告システムが含まれています。物理デバイスにデプロイせずに、Godotエディタ内で広果統合のビジュアルレイアウトを直接テストできます。

詳細については、[エディタでのモック広告のプレビュー](editor_mock_ads.ja.md)を参照してください。

---

## 2. Googleのサンプル広告ユニットの使用

物理デバイスまたはエミュレータでテストを有効にする最も簡単な方法は、Googleの公開テスト広告ユニットを使用することです。

iOSでテストするにはiOSテスト広告ユニットを、AndroidでテストするにはAndroidテスト広告ユニットを使用：

=== "Android"

    | 広告フォーマット     | サンプル広告ユニットID              |
    |----------------------|----------------------------------|
    | バナー               | `ca-app-pub-3940256099942544/6300978111` |
    | インタースティシャル | `ca-app-pub-3940256099942544/1033173712` |
    | リワード             | `ca-app-pub-3940256099942544/5224354917` |
    | リワードインタースティシャル | `ca-app-pub-3940256099942544/5354046379` |

=== "iOS"

    | 広告フォーマット     | サンプル広告ユニットID              |
    |----------------------|----------------------------------|
    | バナー               | `ca-app-pub-3940256099942544/2934735716` |
    | インタースティシャル | `ca-app-pub-3940256099942544/4411468910` |
    | リワード             | `ca-app-pub-3940256099942544/1712485313` |
    | リワードインタースティシャル | `ca-app-pub-3940256099942544/6978759866` |

!!! warning "重要"

    ゲームをアプリストアに公開する前に、これらのテストIDを独自の本番広告ユニットIDに置き換えてください。

## 3. テストデバイスの登録

AndroidエミュレータとiOSシミュレーターは自動的にテストデバイスとして設定されます。

自分の物理デバイスでテストしたい場合は、AdMobコンソールでテストデバイスとして登録できます：
1. AdMobダッシュボードを開く。
2. **設定 -> テストデバイス**に移動。
3. **テストデバイスを追加**をクリックし、デバイスの詳細を入力。
