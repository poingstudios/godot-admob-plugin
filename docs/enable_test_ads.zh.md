# 启用测试广告

!!! note "Godot 3 (v1) 文档"
    本页面适用于 **Godot 3.x**。如需 **Godot 4.2+**，请查看[稳定文档](https://poingstudios.github.io/godot-admob-plugin/stable/)。

在开发期间，使用测试广告至关重要，以避免产生无效流量并冒着AdMob账户被暂停的风险。

---

## 1. 在编辑器中预览模拟广告

插件内置了模拟广告系统。您可以直接在Godot编辑器内测试广告集成的视觉布局，而无需部署到物理设备。

更多详情，请参阅[在编辑器中预览模拟广告](editor_mock_ads.zh.md)。

---

## 2. 使用Google示例广告单元

在物理设备或模拟器上启用测试的最简单方法是使用Google的公开测试广告单元。

在iOS上测试时使用iOS测试广告单元，在Android上测试时使用Android测试广告单元：

=== "Android"

    | 广告格式             | 示例广告单元ID                      |
    |----------------------|----------------------------------------|
    | 横幅                 | `ca-app-pub-3940256099942544/6300978111` |
    | 插页                 | `ca-app-pub-3940256099942544/1033173712` |
    | 激励                 | `ca-app-pub-3940256099942544/5224354917` |
    | 激励插页             | `ca-app-pub-3940256099942544/5354046379` |

=== "iOS"

    | 广告格式             | 示例广告单元ID                      |
    |----------------------|----------------------------------------|
    | 横幅                 | `ca-app-pub-3940256099942544/2934735716` |
    | 插页                 | `ca-app-pub-3940256099942544/4411468910` |
    | 激励                 | `ca-app-pub-3940256099942544/1712485313` |
    | 激励插页             | `ca-app-pub-3940256099942544/6978759866` |

!!! warning "重要"

    将游戏发布到应用商店之前，请记得将这些测试ID替换为您自己的生产广告单元ID。

---

## 3. 注册测试设备

Android模拟器和iOS模拟器会自动配置为测试设备。

如果您想在自己的物理设备上测试，可以在AdMob控制台中将其注册为测试设备：
1. 打开您的AdMob仪表板。
2. 导航到**设置 -> 测试设备**。
3. 点击**添加测试设备**并输入您的设备详情。
