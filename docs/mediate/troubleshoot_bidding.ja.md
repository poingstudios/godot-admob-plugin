# 入札のトラブルシューティング

入札 (Bidding) を使用すると、広告ソースがリアルタイムで広告枠を競い合うことができます。このページでは、一般的な入札の問題に対するトラブルシューティングの手順を説明します。

## 一般的な入札の問題

### 広告が読み込まれない

入札を使用しているときに広告が読み込まれない場合：

1. **アダプターのバージョンの確認**: 最新バージョンの入札アダプターを使用していることを確認します
2. **初期化の検証**: SDK と入札アダプターが正しく初期化されていることを確認します
3. **広告ユニットの設定の確認**: AdMob UI で広告ユニットが入札用に設定されていることを確認します

### 充填率が低い

入札で充填率が低い問題が発生している場合：

1. **広告ソースのステータスの確認**: 広告ソースがアクティブで正しく設定されていることを確認します
2. **フロア価格の確認**: フロア価格が高すぎないか確認します
3. **地域的なカバレッジ**: 一部の広告ソースは、地域的なカバレッジが限られています

### タイムラグ（レイテンシ）の問題

入札によってタイムラグが発生している場合：

1. **ネットワーク状態の確認**: ネットワーク状態が悪いと、入札のタイムラグが増加する可能性があります
2. **アダプターのレイテンシの監視**: 初期化ステータスを使用して、アダプターのレイテンシを確認します
3. **タイムアウト設定の最適化**: 利用可能な場合は、タイムアウト設定を調整します

## デバッグツール

### 広告インスペクタ (Ad Inspector)

入札の問題をデバッグするには、広告インスペクタを使用します。

1. アプリ内で広告インスペクタを有効にします
2. 広告が読み込まれたときに広告インスペクタを起動します
3. 入札ステータスと応答情報を確認します

### ログ

詳細な入札情報を確認するには、ログを有効にします。

=== "GDScript"

    ```gdscript
    # デバッグログを有効にする
    MobileAds.set_debug(true)
    ```

=== "C#"

    ```csharp
    // デバッグログを有効にする
    MobileAds.SetDebug(true);
    ```
